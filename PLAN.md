# 학습 계획

## 범위 — 버릴 것부터 확정

| 다룰 것 | 이유 |
| --- | --- |
| 자료구조 조작 (dict/set/list) | 모든 문제의 전제, **최대 투자** |
| 해시 (카운팅) | 빈도 높고 난이도 낮음 — 가성비 최상 |
| 정렬 + 그리디 | 패턴 단순, Lv.2 다수 커버 |
| 완전탐색 | 구현력만으로 풀림 — 기존 강점 활용 |
| **BFS/DFS** | **신입 코테 최빈출**, 템플릿 암기로 다수 해결 |

| 버릴 것 | 이유 |
| --- | --- |
| DP | 8주 안에 안정화 불가, 맛보기만 |
| 이분탐색 · 투포인터 | 여유 시 선택 |
| 세그먼트 트리 · 펜윅 트리 | 신입 코테 출제 거의 없음, **완전 제외** |
| Lv.3 이상 | 목표 아님 |

## 문제 풀이 루프

타임아웃 실패를 막는 절차 — **구현 전 필수**

| 단계 | 내용 |
| --- | --- |
| 1 | 입력 최대 크기 `N` 확인 |
| 2 | 허용 복잡도 역산 ([PATTERNS.md](PATTERNS.md) 표 참조) |
| 3 | 떠올린 풀이의 복잡도 계산 → 초과 시 **구현 시작 금지** |
| 4 | 구현 |
| 5 | 예제 + 경계값 (`N=1`, 최댓값) 검증 |

**30분 룰** — 30분 막히면 풀이 확인 후 **다음 날 백지 재현**
